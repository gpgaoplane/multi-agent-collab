# Multi-Agent Collab — v0.2.0 Distribution Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Ship `multi-agent-collab` as a cross-agent-discoverable, npm-installable skill. Close the two onboarding gaps (front-door `AGENTS.md`, generic new-agent `--join` flow), add a `SKILL.md` wrapper for in-agent discovery (Claude / OpenCode / Cursor / Antigravity), and package the repo for `npx` invocation.

**Architecture change vs v0.1.0:** still pure-bash scripts + templates. Adds a thin Node.js shim (`bin/cli.js`) for the npm channel and one new top-level `SKILL.md` for agent-native discovery. No runtime dependency changes inside the bootstrap itself.

**Tech Stack:** Bash 4+, POSIX awk, Python 3 (existing fallback), Node.js ≥18 (new, for npm shim only), npm (publish channel).

**Prior art consulted:** `anthropics/skills` (SKILL.md spec), `opencode.ai/docs/skills/` (multi-dir discovery), AGENTS.md standard at agents.md, `Karanjot786/agent-skills-cli` (distribution reference). See parent design doc for rationale.

**Step 3: Write the template — `templates/AGENTS.md`**

This template is the *injected content*, not the whole file. It will be wrapped in markers and merged into any existing `AGENTS.md`.

```markdown
<!-- collab:agents-md:start -->
# Multi-Agent Collaboration

This repo uses the [`multi-agent-collab`](https://github.com/gpgaoplane/multi-agent-collab) skill for coordinating multiple AI coding agents (Claude, Codex, Gemini, and others). Every AI agent working here MUST read these files before taking any action:

1. **`AI_AGENTS.md`** — Shared contract: behavioral rules, onboarding checklist, fan-out routing.
2. **`.collab/INDEX.md`** — File registry. Use delta-read based on `last-updated` timestamps.
3. **`.collab/ROUTING.md`** and **`.collab/PROTOCOL.md`** — End-of-task fan-out matrix and Task Receipt format.
4. **Your own adapter** at `.<agent>/` (or `GEMINI.md` at root for Gemini).

## New-agent self-check (MANDATORY before first action)

If `.collab/agents.d/<your-agent-name>.yml` does NOT exist:

1. **Stop.** Do not proceed with any other work.
2. Ask the user to confirm your agent name (e.g., `opencode`, `cline`, `aider`).
3. Run: `./scripts/collab-init.sh --join <your-agent-name>`
4. That command creates your adapter, memory dir, work log, and descriptor, and registers them in `.collab/INDEX.md`.
5. Then proceed with the Onboarding Checklist in `AI_AGENTS.md`.

Skipping this step means you will step on other agents' files without logging your work. Do not skip.
<!-- collab:agents-md:end -->
```

**Step 4: Wire it into `collab-init.sh`.**

Modify `setup_shared()` to emit an AGENTS.md from the template if absent, and inject the managed section if present:

```bash
# After existing setup_shared body, before the final brace:
if [[ ! -f AGENTS.md ]]; then
  # Fresh: copy template as the whole file.
  copy_file "$TEMPLATES/AGENTS.md" "AGENTS.md"
else
  # Existing: ensure managed markers exist; if present, replace managed content; if absent, append.

fi
```

Add a new helper near `refresh_managed_sections`:

```bash
inject_agents_md_section() {
  local target="AGENTS.md"
  local template="$TEMPLATES/AGENTS.md"

  if merge_has_section "$target" "agents-md"; then
    # Refresh managed content only.
    local new_content
    new_content=$(awk -v start="<!-- collab:agents-md:start -->" -v end="<!-- collab:agents-md:end -->" '
      $0 == start { in_sec = 1; next }
      $0 == end { in_sec = 0; next }
      in_sec { print }
    ' "$template")
    if [[ $DRY_RUN -eq 1 ]]; then
      say "would refresh AGENTS.md managed section"
    else
      merge_replace_section "$target" "agents-md" "$new_content"
    fi
  else
    # Markers absent — append the whole managed block (with its markers) to the end of the file.
    if [[ $DRY_RUN -eq 1 ]]; then
      say "would append managed section to AGENTS.md"
    else
      {
        echo
        cat "$template"
      } >> "$target"
    fi
  fi
}
```

Also update `re_init_shared()` to call `inject_agents_md_section` when AGENTS.md exists (so re-init refreshes managed content).

```bash
# join_agent <name>
# Implements the three-tier descriptor lookup and bootstrap.
join_agent() {
  local name="$1"
  local upper="$(echo "$name" | tr '[:lower:]' '[:upper:]')"
  local display="$(echo "${name:0:1}" | tr '[:lower:]' '[:upper:]')${name:1}"

  local shipped="$TEMPLATES/agents.d/${name}.yml"
  local user="$PWD/.collab/agents.d/${name}.yml"
  local dest=".collab/agents.d/${name}.yml"

  if [[ -f "$user" ]]; then
    say "join: using existing descriptor at $user"
  elif [[ -f "$shipped" ]]; then
    say "join: installing shipped descriptor for $name"
    if [[ $DRY_RUN -eq 0 ]]; then
      mkdir -p ".collab/agents.d"
      cp "$shipped" "$dest"
    fi
  else
    say "join: rendering generic descriptor for unknown agent $name"
    local generic="$TEMPLATES/agents.d/_generic.yml"
    if [[ ! -f "$generic" ]]; then
      echo "join: generic template missing at $generic" >&2
      return 1
    fi
    if [[ $DRY_RUN -eq 0 ]]; then
      mkdir -p ".collab/agents.d"
      local content
      content=$(cat "$generic")
      content="${content//\{\{AGENT_NAME\}\}/$name}"
      content="${content//\{\{AGENT_DISPLAY\}\}/$display}"
      content="${content//\{\{AGENT_UPPER\}\}/$upper}"
      printf '%s' "$content" > "$dest"
    fi
  fi

  bootstrap_agent "$dest"
}
```

## Task 4: Migration script for v0.1.0 → v0.2.0 users

**Files:**
- Create: `scripts/migrations/0.1.0-to-0.2.0.sh`

Existing v0.1.0 users who run the v0.2.0 `collab-init.sh` hit `detect_mode → upgrade`. The upgrade handler (v0.1.0 Task 17) looks for `scripts/migrations/<from>-to-<to>.sh` and runs it. This script brings a v0.1.0 repo up to v0.2.0 shape.

**What needs to change:**
- `AGENTS.md` must exist (created if absent, markers injected if present).
- `templates/agents.d/_generic.yml` available for `--join` fallback — but this is in the skill templates, not the target repo. No target-repo migration needed.
- `.collab/VERSION` already bumped automatically by upgrade-mode's final `echo "$shipped" > .collab/VERSION`.

So the migration's only real job is invoking `inject_agents_md_section`. Everything else is picked up by the subsequent `re_init_shared()` call.

**Step 1: Write the script.**

```bash
#!/usr/bin/env bash
# Migration: 0.1.0 → 0.2.0
# - Adds AGENTS.md front door (merge-aware).
# - No other target-repo state changes required.
set -euo pipefail

HERE="$(cd "$(dirname "$0")" && pwd)"
SCRIPTS="$(cd "$HERE/.." && pwd)"
SKILL_ROOT="$(cd "$SCRIPTS/.." && pwd)"
TEMPLATES="$SKILL_ROOT/templates"

source "$SCRIPTS/lib/merge.sh"

echo "Migrating v0.1.0 → v0.2.0..."

# Re-define inject helper locally (the main collab-init sources this behavior
# but we want the migration to be self-contained so it can be re-run manually).
inject_agents_md_section() {
  local target="AGENTS.md"
  local template="$TEMPLATES/AGENTS.md"

  if [[ ! -f "$target" ]]; then
    cp "$template" "$target"
    echo "  created AGENTS.md"
    return
  fi

  if merge_has_section "$target" "agents-md"; then
    local new_content
    new_content=$(awk -v start="<!-- collab:agents-md:start -->" -v end="<!-- collab:agents-md:end -->" '
      $0 == start { in_sec = 1; next }
      $0 == end { in_sec = 0; next }
      in_sec { print }
    ' "$template")
    merge_replace_section "$target" "agents-md" "$new_content"
    echo "  refreshed managed AGENTS.md section"
  else
    {
      echo
      cat "$template"
    } >> "$target"
    echo "  appended managed AGENTS.md section to existing file"
  fi
}

inject_agents_md_section

echo "Migration 0.1.0 → 0.2.0 complete."
```

## Task 6: Top-level `SKILL.md` wrapper

**Files:**
- Create: `SKILL.md` (at repo root)

**Purpose:** when a user clones this repo into `~/.claude/skills/multi-agent-collab/` or `~/.agents/skills/multi-agent-collab/`, any SKILL.md-aware agent (Claude, OpenCode, Cursor) discovers the skill via its frontmatter and loads this file when the user's intent matches.

**Key design rules** (from retrospect):
- Description must gate irrelevant loads — state explicitly "Invoke only when..."
- Body must open with the state-check guard — agents that double-load the skill self-abort.
- Primary invocation uses `npx`; fallback uses skill directory absolute path.

**Step 1: Write `SKILL.md`.**

```markdown
# multi-agent-collab — bootstrap

## Step 1: Check current state (MANDATORY — do this before anything else)

Run in the user's current repo:

    test -f .collab/VERSION && cat .collab/VERSION || echo "not installed"

**If the output is a version string (e.g., `0.2.0`):**
The repo is already bootstrapped. STOP. Tell the user:
"multi-agent-collab is already installed (version X.Y.Z)."
Do NOT re-run bootstrap. Only continue to Step 3 if the user has
explicitly asked to add another agent.

**If the output is `not installed`:**
Proceed to Step 2.

## Step 2: Fresh bootstrap

Preferred invocation (works from any agent, no clone required):

    npx @gpgaoplane/multi-agent-collab init

Fallback (if `npx` is unavailable or the skill is installed as a
local clone at `$SKILL_DIR`):

    bash "$SKILL_DIR/scripts/collab-init.sh"

Both paths are idempotent. User content outside `<!-- collab:...:start/end -->`
markers is preserved on re-run.

After bootstrap, the target repo will contain:

- `AI_AGENTS.md` — shared contract read by every agent
- `AGENTS.md` — cross-agent front door pointing at AI_AGENTS.md
- `.collab/` — INDEX, ACTIVE, ROUTING, PROTOCOL, descriptors
- `.claude/`, `.codex/`, `GEMINI.md`, `.gemini/` — per-agent adapters and memory
- `docs/agents/{claude,codex,gemini}.md` — work logs

## Step 3: Add a new agent (only when user explicitly requests)

Run in the target repo:

    npx @gpgaoplane/multi-agent-collab join <agent-name>

Or with a local clone:

    bash "$SKILL_DIR/scripts/collab-init.sh" --join <agent-name>

The `--join` flag performs a three-tier descriptor lookup:
1. Known agent (claude / codex / gemini) → uses shipped descriptor.
2. Existing user descriptor at `.collab/agents.d/<name>.yml` → uses as-is.
3. Unknown → renders generic template with sensible defaults.

For unknown agents with non-standard conventions (e.g., adapter file
at repo root instead of `.{name}/`), the user should edit
`.collab/agents.d/<name>.yml` after generation, then re-run `--join`.

## What this skill does NOT do

- It does NOT install itself repeatedly. The state check in Step 1
  short-circuits on already-installed repos.
- It does NOT modify files outside the managed marker sections.
- It does NOT enforce rules. Runtime behavioral conventions live in
  `AI_AGENTS.md` (produced at bootstrap), read by every agent every
  session. This skill file is the installer only.

## Related files in this skill directory

- `docs/design.md` — full rationale and design philosophy
- `docs/plans/2026-04-22-multi-agent-collab-implementation.md` — v0.1.0 build plan
- `docs/plans/2026-04-22-v0.2.0-distribution.md` — this release's plan
- `README.md` — end-user install and usage
```

```javascript
#!/usr/bin/env node
// Thin Node shim that spawns the bundled bash bootstrap.
// Hard requirement: bash must be on PATH. On Windows, users need Git Bash or WSL.

const { spawn } = require('child_process');
const path = require('path');
const fs = require('fs');

const USAGE = `Usage: multi-agent-collab <command> [args]

Commands:
  init                  Bootstrap multi-agent collab in the current repo
  join <name>           Add a new agent (claude/codex/gemini or any name)
  check                 Audit INDEX vs filesystem
  archive <path>        Archive a file
  register <path>       Register a file in INDEX
  --help, -h            Show this help

Requires bash. On Windows, install Git for Windows (https://git-scm.com/download/win)
or use WSL.`;

function main() {
  const argv = process.argv.slice(2);
  if (argv.length === 0 || argv[0] === '--help' || argv[0] === '-h') {
    console.log(USAGE);
    process.exit(argv.length === 0 ? 1 : 0);
  }

  const cmd = argv[0];
  const rest = argv.slice(1);

  // Resolve the bundled scripts dir relative to this file.
  const scriptsDir = path.resolve(__dirname, '..', 'scripts');

  let scriptPath;
  let scriptArgs = [];

  switch (cmd) {
    case 'init':
      scriptPath = path.join(scriptsDir, 'collab-init.sh');
      break;
    case 'join':
      if (rest.length === 0) {
        console.error('join: missing <name> argument');
        process.exit(1);
      }
      scriptPath = path.join(scriptsDir, 'collab-init.sh');
      scriptArgs = ['--join', rest[0]];
      break;
    case 'check':
      scriptPath = path.join(scriptsDir, 'collab-check.sh');
      break;
    case 'archive':
      if (rest.length === 0) {
        console.error('archive: missing <path> argument');
        process.exit(1);
      }
      scriptPath = path.join(scriptsDir, 'collab-archive.sh');
      scriptArgs = [rest[0]];
      break;
    case 'register':
      if (rest.length === 0) {
        console.error('register: missing <path> argument');
        process.exit(1);
      }
      scriptPath = path.join(scriptsDir, 'collab-register.sh');
      scriptArgs = [rest[0]];
      break;
    default:
      console.error(`Unknown command: ${cmd}`);
      console.error(USAGE);
      process.exit(1);
  }

  if (!fs.existsSync(scriptPath)) {
    console.error(`Bundled script missing: ${scriptPath}`);
    console.error('This is a package bug. Reinstall or report.');
    process.exit(1);
  }

  // Detect bash. On Windows, `bash` resolves to Git Bash if installed.
  const bash = process.platform === 'win32' ? 'bash.exe' : 'bash';
  const child = spawn(bash, [scriptPath, ...scriptArgs], {
    stdio: 'inherit',
    shell: false,
  });

  child.on('error', (err) => {
    if (err.code === 'ENOENT') {
      console.error('bash not found on PATH.');
      console.error('  On Windows: install Git for Windows (https://git-scm.com/download/win)');
      console.error('  On Linux/macOS: install bash via your package manager');
      process.exit(127);
    }
    throw err;
  });

  child.on('exit', (code) => {
    process.exit(code ?? 0);
  });
}

main();
```

## Watch-outs during execution

- **Task 1's `inject_agents_md_section`** is called from both `setup_shared` and `re_init_shared`. Verify the re-init path does not duplicate the managed block — the existing `merge_replace_section` in `scripts/lib/merge.sh` already handles this, but double-check by running the task's test twice.
- **Task 3's three-tier lookup** must check `user` before `shipped` to honor user customizations. If you implement the other order, a user's descriptor edit is silently overwritten on re-run. The test's Case 4 (gemini) catches this only if `.collab/agents.d/gemini.yml` is deleted first; be explicit about the order in the code.
- **Task 9's bash detection on Windows** — `spawn('bash.exe', ...)` works only if Git Bash is on PATH. If Git is installed but PATH isn't configured, the error path fires. That's the documented behavior; do not add PATH-search fallbacks (too fragile, hides the real fix).
- **Task 10's npm test** depends on `npm` being in PATH during `./tests/run-all.sh`. If a contributor runs tests without Node installed, this test fails hard. Consider adding a skip at the top: `command -v npm >/dev/null || { echo "skip: npm not found"; exit 0; }`.
- **Task 11's `npm publish`** is irreversible (cannot unpublish after 72 hours). Triple-check the `version` field in `package.json` and the git tag agree before running `publish`.
