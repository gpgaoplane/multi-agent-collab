# v0.4.0 Plan — Bootstrap Correctness, Log Rotation, Vocabulary Symmetry

> **Status: shipped 2026-04-26.** Tag `v0.4.0` pushed; published as `@gpgaoplane/multi-agent-collab@0.4.0` on npm at 2026-04-26T22:03:57Z (GitHub Actions run `24968159835` succeeded in 1m3s). Final commit on origin/main: `331c0ff`. Run summary in `docs/plans/2026-04-26-autonomous-run-status.md`.

## 1. Theme

Three load-bearing fixes:

1. **Bootstrap correctness** (Q2) — `init` should install only the calling agent, not all three.
2. **Log durability** (Q1, partial Q7) — append-only work logs grow unbounded; introduce rotation with Receipts as the lossless preservation contract.
3. **Vocabulary symmetry** (Q4) — sender-side handoff phrases are missing; pickup ritual is asymmetric.

Plus targeted fixes for commit cadence (Q3), post-compact persistence (Q6), upgrade communication (Q8), and a single highest-leverage token cost mitigation (Q7) — trimming `AI_AGENTS.md` itself. Other token-cost tooling deferred to v0.5.0 because Q1's rotation eliminates the dominant cost driver.

## 2. Scope decisions

- Q2: every fresh `init` materializes only the calling agent. Others arrive via `join`. Behavior change with migration.
- Q1: 300-line default rotation threshold, configurable. Receipts + open handoff blocks preserved verbatim.
- Q7: keep only `AI_AGENTS.md` trim. Defer `collab-check --token-budget` to v0.5.0.
- Original deferred items: auto-install hooks, native Windows path, presence-required mode — kept deferred. `collab-register --type/--owner` (H1) and `--to-any` handoff coverage (C5) folded in as small adjacent items.
- A1 detection failure path: **hard-fail** with explicit guidance. No silent default to `claude`.

## 3. Group A — Bootstrap installs only the calling agent (Q2)

**Why first:** correctness bug; everything else assumes correct adapter installation.

- **A1.** Detection ladder in `collab-init.sh`, applied in this exact precedence:
  1. `--agent <name>` flag (explicit user/wrapper choice).
  2. `$COLLAB_AGENT` env override.
  3. Detection probe — documented env-var list per agent: Claude (`CLAUDE_CODE_*`, `CLAUDECODE`, `ANTHROPIC_*`), Codex (`CODEX_HOME`, `CODEX_CLI`), Gemini (`GEMINI_*`, `GOOGLE_AI_*`).
  4. **Hard-fail** with one-line message: `collab-init: cannot detect calling agent. Re-run with --agent claude|codex|gemini, or set COLLAB_AGENT.`
  No silent default. Document the precedence in README and SKILL.md.
- **A2.** Stop copying all `templates/agents.d/*.yml` into `.collab/agents.d/` during `setup_shared`. Only the calling agent's descriptor lands there. Other shipped descriptors stay in the npm package, used only by `join`.
- **A3.** `join <name>` becomes the sole way to add agents. Document prominently in README, SKILL.md, AI_AGENTS.md.
- **A4.** Migration `scripts/migrations/0.3.0-to-0.4.0.sh`: detect agents whose work logs have only the seed entry (no real activity), prompt to remove their adapter dirs + descriptors. Default: prompt; `--non-interactive` keeps everything (conservative).
- **A5.** Tests:
  - claude-only init produces no `.codex/`, `.gemini/`, or `GEMINI.md`.
  - codex-only init produces no `.claude/` or `.gemini/`.
  - `join codex` after claude-only init works idempotently.
  - Migration prunes empty agents but preserves agents with real activity.
  - **Hard-fail path:** `init` without detection and without `--agent` exits non-zero with the documented message.
  - **Dynamic adapter-table assertion:** AI_AGENTS.md's `collab:current-adapters` table contains exactly one row after claude-only init, two rows after `join codex`.
- **A6.** Render `<!-- collab:current-adapters -->` in `templates/AI_AGENTS.md` dynamically from `.collab/agents.d/*.yml` on init, on `join`, and on migration. The current hardcoded three-row table at lines 27–35 of the template becomes a templated section. Re-render any time descriptor set changes.

**Dependencies:** none.

## 4. Group B — Work-log rotation (Q1, partial Q7)

- **B1.** Add to `templates/config.yml` and `.collab/config.yml`:
  - `rotate_at_lines: 300` (threshold)
  - `rotate_keep_recent: 8` (live entry count after rotation)
- **B2.** New `scripts/collab-rotate-log.sh <agent>`:
  - Reads agent's `log_path` from descriptor.
  - **Splits entries on `^## ` followed by an ISO-8601 timestamp pattern** (matches the seed entry header format). Other `## ` headings inside an entry (e.g. `## Files`, `## Notes`) are NOT entry boundaries.
  - **Normalizes line endings to LF before counting** to neutralize CRLF on Windows.
  - Keeps last `rotate_keep_recent` entries verbatim.
  - Keeps all open handoff blocks verbatim regardless of age.
  - Writes archive `.collab/archive/agents/<agent>-<YYYYMMDD>.md` with frontmatter `status: archived`.
  - Replaces archived entries in live log with one-line Receipt summaries (date + "Updates fanned out" list).
  - Registers archive in INDEX, bumps live log's `last-updated`.
- **B3.** New marker sections in `templates/work-log-seed.md`:
  - `<!-- collab:log-live:start/end -->` wraps live entries.
  - `<!-- collab:log-archived-summary:start/end -->` wraps the one-liner summaries.
  - Backward-compatible: existing logs without markers get them on first rotation.
- **B4.** `collab-check` warns when any managed file exceeds `rotate_at_lines`. Suggests the rotate command. Silent if under threshold.
- **B5.** Tests:
  - Rotation preserves all Receipts (including from archived entries — present as one-liners).
  - Rotation preserves all open handoff blocks verbatim.
  - **Idempotence:** run rotation twice on a log already at threshold; second run is a no-op (no growth in archived-summary section).
  - Archive file is INDEX-registered with `status: archived`.
  - Threshold is read from config.
  - **CRLF fixture:** rotation works on a log with `\r\n` line endings without miscounting.
  - **Subsection fixture:** an entry containing `## Files` and `## Notes` is not split mid-entry.

**Dependencies:** can run parallel to A. A4 migration is sequencing-coupled to A's other tasks but B itself is not.

## 5. Group C — Handoff vocabulary & receiver pickup (Q4)

- **C1. Sender vocabulary** — add to `templates/collab/PROTOCOL.md` and AI_AGENTS.md `behavioral-rules`:

  > **User says** "wrap up for handoff", "prepare handoff to <agent>", "hand it off to <agent>", "tag out to <agent>".
  >
  > **Agent does:** run End-of-Task Protocol → emit Receipt → run `./scripts/collab-handoff.sh <to-agent> --from <self> --message "…" --files "…"` → confirm with the printed handoff ID.

- **C2. Receiver vocabulary** — expand existing list in PROTOCOL.md line 73:

  > **User says** "take the baton", "pick up handoff", "take over from <agent>", "you're up".
  >
  > **Agent does:** run `./scripts/collab-catchup.sh preview --agent <self> --handoff` → if exactly one open block targets you, run pickup automatically; if multiple, ask the user which.

- **C3.** New verb `collab-handoff pickup <id> --from <self>`:
  - Reads block by ID across all agent logs.
  - **Prints a ready-to-paste summary block** ("What I did / Files / Branch / Open questions") to stdout. The receiver agent decides whether to update its own `state.md` manually (preserves the "do not edit another agent's memory" rule and avoids typo-induced cross-agent writes).
  - Stamps `picked-up: <ISO-8601>` metadata onto the handoff block in-place.
  - Status stays `open` until validation; `close` follows.
  - Bumps INDEX timestamps for the modified log.

- **C4.** Document `close <id>` and `cancel <id>` as today; both should follow validation. Sender phrases trigger create; receiver phrases trigger pickup; explicit "validated, closing handoff X" triggers close.

- **C5.** Verify `--to-any`: the existing regex `to ~ / any$/` in `collab-catchup.sh` ~line 73 already supports it. Two tasks: (a) document the literal `any` token in PROTOCOL.md so users know the magic word, (b) add the positive test that `collab-handoff any --from <sender>` produces a block visible to every agent's `--handoff` preview.

- **C6.** Tests:
  - Pickup prints expected block to stdout.
  - Pickup is idempotent (re-running on same id is a no-op apart from updated `picked-up` timestamp).
  - Close after pickup keeps the `picked-up` metadata.
  - Vocabulary phrases are documented in canonical PROTOCOL.md location.
  - **Positive `to: any` test:** a handoff written with `to: any` is surfaced by every agent's `catchup --handoff` preview.

### Follow-ups captured 2026-04-26 (after C1-C6 closed)

- **C7. Log rotation user vocabulary.** Add a "Log rotation vocabulary" section to `templates/collab/PROTOCOL.md` (sibling to the existing user-vocabulary section). Phrases the user can speak to trigger rotation:
  - "the log is getting long, let's compact it"
  - "rotate the log" / "rotate my log"
  - "summarize old entries"
  - "trim my work log" / "trim the log"
  - "archive old entries"
  - "compact the work log"

  **Sanity rule:** never rotate another agent's log on the user's verbal request alone — rewriting another agent's memory violates cross-agent courtesy. If the user means another agent, ask them to switch to that agent.

- **C8. Framework upgrade user vocabulary.** Add an "Upgrade vocabulary" section to PROTOCOL.md. Phrases the user can speak to trigger framework update:

  - "get the latest version" / "pull the latest"
  - "check for updates and apply"
  - "update multi-agent-collab"
  - "is there a new version" (prompts a non-destructive check first)

  **Agent action:**
  1. Run `bash scripts/collab-check.sh` first to see if a newer version exists (uses the existing update advisory).
  2. If newer version exists, run `npx @gpgaoplane/multi-agent-collab init` (or local-clone equivalent).

## 10b. Group M — Marker preservation + migration safety (added 2026-04-26)

**Why added:** the user asked for an explicit map of what's preserved vs lost during migration, and approved all six optimizations to maximize preservation without overhauling the migration system. These are layered improvements to the existing system; none replace it.

- **M1. Inline warning comments inside every framework-managed marker block.** Right after each `<!-- collab:NAME:start -->` line, insert a one-line HTML comment: `<!-- WARNING: framework-managed; edit OUTSIDE this block, not inside -->`. Apply to every marker in every templated file (AI_AGENTS.md, AGENTS.md, work-log-seed.md, INDEX.md, ACTIVE.md). Tests: each marker block contains the warning; refresh preserves it.

- **M2. Pre-migration cleanliness check in `collab-init.sh`.** When entering upgrade mode, run `git status --porcelain` first. If the working tree has tracked changes, print a warning + require `--force-dirty` to proceed. Cost: avoids the most common silent loss (migration changes mixing with in-flight work). Tests: dirty tree blocks upgrade by default, `--force-dirty` overrides, clean tree proceeds normally, untracked files alone don't block.

- **M3. Backup + restore flags.**
  - `collab-init --backup` (or always-on during upgrade): before any migration, snapshot all framework-managed files to `.collab/.backup-<from>-to-<to>-<YYYYMMDDHHMMSS>/` preserving directory structure.
  - `collab-init --restore <backup-id>` (or `latest`): copy files from a backup back to live locations.
  - The backup directory has frontmatter `status: backup` so agents skip it.
  - Tests: backup created during upgrade, file equality vs original, restore works on clean and migrated states, idempotence of multiple backups.

- **M4. Migration `--diff` flag.** Show what each migration would change without applying. Mode: `collab-init --diff` exits without modifying anything but prints a unified diff per file the migration would touch. Tests: --diff doesn't modify files, output includes file paths and diff hunks, --diff after upgrade shows nothing (state is at shipped).

- **M5. Loud per-migration logging.** Each migration prints structured `BEFORE: file X had N lines, M markers | AFTER: O lines, P markers, sections refreshed: [...]` lines. Captured into UPGRADE_NOTES.md alongside the existing `>>> Upgrade summary` block. Tests: log lines appear in stdout, structured fields parseable.

- **M6. "How to safely customize managed files" docs section.** Add a clear paragraph to AI_AGENTS.md (in a new managed section `collab:customization-guide`) explaining the marker convention: edit outside markers, not inside. Show one good example, one bad example. Reference: this section is itself inside markers and is therefore managed — its CONTENT illustrates the rule. Tests: section exists, examples present.

**Dependencies:** M1 and M6 are docs-only and can run anywhere. M2 depends on collab-init.sh changes; runs before M3/M4/M5. M3 should land before M4 since --diff complements --backup. M5 modifies migration scripts; can run independently.

**Risks/gotchas:**
- M2 false positives: users who legitimately have unrelated working-tree changes will be slowed. The `--force-dirty` escape hatch mitigates.
- M3 backup directory bloats `.collab/`. Document a `--prune-backups` flag (defer to v0.5.0 if scope is tight) or recommend `git clean .collab/.backup-*` periodically.
- M4 --diff invocation that ALSO has migration scripts modifying state outside the marker system would not be reflected. Migrations today don't do this, but a future migration could; document that `--diff` shows declarative changes only.

## 13. Risks and gotchas

- **A1 detection brittleness.** Env-var heuristics are not stable contracts across CLI versions. Hard-fail with `--agent` guidance is the mitigation. Document the var list and accept that future agents may need additions.
- **A4 migration aggressiveness.** Default `--non-interactive` to keep-everything; only prompt mode prunes.
- **B2 entry-boundary regex.** Splitting on `^## ` plus ISO-8601 timestamp is the contract. Any future template change to entry headers must update the regex; mark this explicitly in `collab-rotate-log.sh`.
- **C3 cross-agent write avoided.** Pickup prints to stdout instead of mutating `state.md` directly; agent owns its own memory writes. Aligns with existing rule.
- **F3 race.** Explicit ack via `--ack-upgrade` avoids the side-effect-of-reading race.
- **G2 line cap.** 100 lines after all additions; if it busts the cap, move sections explicitly listed in G1 first; raise cap only as a last resort and document why.
